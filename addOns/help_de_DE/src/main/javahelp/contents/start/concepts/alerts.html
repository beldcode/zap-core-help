Warnungen

Eine Warnung ist eine potentielle Sicherheitslücke bezogen auf eine spezielle Anfrage. Eine Anfrage kann mehr als nur eine Warnung beinhalten.

Warnungen werden in der UI durch ein Flag angezeigt, welches das Vorhandensein einer Lücke signalisiert:
      Hoch
      Mittel
      Niedrig
      Informatorisch
      Falsch Positiv

Warnungen können durch zahlreiche ZAP Komponenten ausgeöst werden, z.B. active scanning, passive scanning, Skripen, Add-Ons (Erweiterungen), oder selbst erstellten Warnungen, welche mittels Add Alert dialog erzeugt werden können.

Warnungen werden im History tab gelistet mit einem Flag, welches die Warnung mit dem höchsten Sicherheitsrisiko kennzeichnet. Sämtliche Warnungen werden im Alerts tab aufgeführt. Eine Aufführung der absoluten Häufigkeiten der einzelnen Warnungen geordnet nach Art der Bedrohung lässt sich in footer finden.

Alert overrides

Warnungen, welche von ZAP ausgegeben werden, enthalten sowohl generische als auch spezifische Informationen bezüglich der Ereignisse. Die spezifischen Informationen beziehen sich direkt auf das auslösende Ereignis, z.B. die URL und darin enthaltene Parameter. Die generischen Informationen beinhalten Dinge, wie eine Beschreibung und weiterführende Links zu Online-Ressourcen. Man kann die generischen Informationen mittels einer 'alert override' Konfigurationsdatei hinzufügen oder ersetzen. Dies erlaubt die Einbeziehung von Informationen, die spezifisch für ein Unternehmen sind, wie z.B. Unternehmens-Policies, interne Links oder Ratschläge zu speziellen Technologien, die im Unternehmen Anwendung finden.

Eine 'alert override' Konfigurationsdatei ist eine UTF-8 formatierte Datei, welche genau die Informationen enthält, die man ändern möchte. Zeilen, die mit '#' beginnen gelten als Kommentare und werden ignoriert.

Das Format ist: 

<alertid>.<property> = [ + | - ] <your information>
Folgende Eigenschaften werden unterstütz:
  • Name
  • Beschreibung
  • Lösung
  • sonstige Informationen
  • Referenz
For example 
# 40012 = Reflected XSS
40012.Lösung = Folgen Sie unseren unternehmensspezifischen Richtlinien unter http://internet.example.com/xss.html
Falls der Wert mit '+' beginnt, wird er zu bestehenden Informationen hinzugefügt.
Falls er mit '-' beginnt, wird er bestehenden Informationen vorangestellt.
Falls er weder mit '+' noch mit '-' beginnt, ersetzt er die bestehenden Informationen.

Die 'alert override' Konfigurationsdatei kann mittels einer API, einem Options Alert screen oder der command line Option 

-config alert.overridesFilename=<filename>
spezifiziert werden
